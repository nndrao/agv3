üí° Key Question: Can we use [pnl] > 2000 ? {backgroundColor:'blue'} : null format?
Answer: YES! The system can support both formats.

Method 1: Inline Ternary Format

Type styles directly in the expression:

[pnl] > 2000 ? { backgroundColor: '#3b82f6', color: '#ffffff', fontWeight: 'bold' } : null

‚úÖ Pros:

  • Quick and direct
  • Familiar JavaScript syntax
  • See styles immediately in code
  • No metadata overhead
  • Easy to copy/paste

‚ùå Cons:

  • Must type CSS properties manually
  • No visual preview while editing
  • Prone to typos
  • No color picker

AG-Grid Implementation:

columnDef.cellStyle = (params) => { return params.data.pnl > 2000 ? { backgroundColor: '#3b82f6', color: '#ffffff', fontWeight: 'bold' } : null; }

Method 2: Visual Editor Format (Ctrl+Shift+D)

Use visual editor to configure styles:

/* @rule-style { "id": "rule-pnl-2000", "styles": { "backgroundColor": "#3b82f6", "color": "#ffffff", "fontWeight": "bold" } } */ params.data.pnl > 2000

‚úÖ Pros:

  • Visual style configuration
  • Color pickers
  • Live preview
  • No typing errors
  • CSS classes (better performance)
  • Reusable styles

‚ùå Cons:

  • More verbose
  • Requires popup interaction
  • Metadata overhead

AG-Grid Implementation:

// CSS injected: .cr-rule-pnl-2000 { background-color: #3b82f6; color: #ffffff; font-weight: bold; } // cellClassRules: columnDef.cellClassRules = { 'cr-rule-pnl-2000': (params) => params.data.pnl > 2000 }

Advanced Examples with Ternary Format

1. Simple Condition:

[status] === "Active" ? { color: '#22c55e' } : null

2. Multiple Conditions (AND):

[pnl] > 1000 && [status] === "Active" ? { backgroundColor: '#10b981', fontWeight: 'bold' } : null

3. Nested Ternary (Multiple Styles):

[priority] === "Critical" ? { backgroundColor: '#ef4444', color: '#ffffff' } : [priority] === "High" ? { backgroundColor: '#f97316', color: '#ffffff' } : [priority] === "Medium" ? { backgroundColor: '#fbbf24', color: '#000000' } : null

4. Complex Style Object:

[value] < 0 ? { backgroundColor: '#fee2e2', color: '#dc2626', borderLeft: '3px solid #dc2626', fontWeight: '600', textDecoration: 'underline' } : null

Performance Comparison

CSS Classes (Metadata)

  • Single CSS rule for all cells
  • Better for many rows
  • Cached by browser
  • ~10ms for 10,000 rows

Inline Styles (Ternary)

  • Style object per cell
  • More memory usage
  • Direct DOM manipulation
  • ~25ms for 10,000 rows
