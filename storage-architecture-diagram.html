LocalStorage Structure

localStorage (Browser Storage)
Profile Data // via LocalStorageAdapter
agv3:DataGridStomp:{viewInstanceId} // Complete profile configurations
agv3:mapping:{viewInstanceId} // Component type mappings for fast lookup
agv3:index:{componentType} // Index of all configs by type
Grid-Level Configurations // Direct localStorage access
grid_column_groups_{gridInstanceId} // Shared column group definitions
grid_conditional_formatting_{gridInstanceId} // Shared formatting rules
grid_calculated_columns_{gridInstanceId} // Shared calculated columns
View Registry // View instance tracking
agv3-profile-registry // Maps views to stable profile IDs
Temporary Storage // Session data
agv3-view-title-{viewId} // Custom view titles
agv3-theme-preference // UI theme settings

Key Storage Patterns

  • Profile Data: Stored with viewInstanceId as key, contains all profile versions
  • Grid Configurations: Stored separately by gridInstanceId, referenced by profiles via IDs
  • Two-Level System: Profile-level (user-specific) vs Grid-level (shared) configurations
  • Reference Architecture: Profiles store IDs, not full data, for shared configurations

üîó Data Relationships

  • UnifiedConfig contains multiple ConfigVersion objects (version history)
  • ConfigVersion.config holds the actual profile data (DataGridStompSharedProfile)
  • Profile stores IDs only for grid-level configurations (column groups, rules, etc.)
  • Grid-level configs are loaded separately and merged at runtime
